Production Deployment Guide

Step-by-step guide to deploy Goodly to production

1 Prerequisites

Accounts & Tools Needed

• Google Cloud account (Cloud Run + Cloud Build)

• Vercel account (frontend hosting)

• MongoDB Atlas account (database)

• Stripe account (payments)

• Resend account (email)

• Google AI Studio account (Gemini API key)

• Domain: searchgoodly.com (or your custom domain)

• gcloud CLI installed locally

3 Deploy Backend (Cloud Run)

Using Cloud Build

The project includes cloudbuild.yaml for automated deployment:

# Submit build gcloud builds submit --config cloudbuild.yaml # Or deploy manually gcloud run deploy goodly-api \ --source . \ --region us-central1 \ --allow-unauthenticated \ --set-env-vars="MONGO_URL=...,JWT_SECRET=...,..."

After deployment, note the Cloud Run URL (e.g., your-service-uc.a.run.app).

4 Deploy Frontend (Vercel)

Vercel Deployment

# Set environment variable in Vercel REACT_APP_BACKEND_URL=https://your-service-uc.a.run.app # Deploy cd frontend vercel --prod

The vercel.json rewrites /api/* to the Cloud Run backend. Vercel handles SSL automatically.

5 Configure DNS

Domain Setup

• Point searchgoodly.com to Vercel (CNAME cname.vercel-dns.com)

• Point www.searchgoodly.com to Vercel (CNAME)

• Point api.searchgoodly.com to Cloud Run (via Google Cloud Load Balancer or custom domain mapping)

• Wait for DNS propagation (up to 48 hours)

6 Configure Stripe Webhooks

Stripe Dashboard Setup

• Go to Stripe Dashboard → Webhooks

• Add endpoint: https://api.searchgoodly.com/api/billing/webhook

• Select events: checkout.session.completed, customer.subscription.updated, customer.subscription.deleted

• Copy the webhook signing secret → set as STRIPE_WEBHOOK_SECRET

7 Verify Deployment

Health Checks

# Backend health curl https://api.searchgoodly.com/api/health # Frontend curl -I https://searchgoodly.com # Status page open https://searchgoodly.com/status

Production Readiness Checklist

○ MongoDB Atlas cluster provisioned
○ All env vars set in Cloud Run
○ Stripe webhooks configured
○ Resend domain verified
○ DNS records configured
○ SSL certificates active
○ Sentry DSN configured
○ Google Analytics ID set
○ Health endpoint returns 200
○ Status page shows all green
○ Stripe test payment successful
○ Email delivery verified
