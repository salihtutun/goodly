Security Overview

Defense-in-depth security architecture — 6 layers of protection

Layer 1: Transport Security

TLS HTTPS Everywhere

• All traffic encrypted with TLS 1.2+ (enforced by Cloud Run and Vercel)

• HSTS header: max-age=31536000; includeSubDomains

• Automatic HTTP → HTTPS redirect

• SSL certificates managed by Google Cloud and Vercel

Layer 2: Authentication

JWT Token-Based Auth

• JWT tokens stored in HttpOnly, Secure, SameSite=Lax cookies

• No localStorage tokens — prevents XSS token theft

• Token expiry: configurable (default 30 days)

• Password hashing: bcrypt with appropriate work factor

• Password reset: time-limited tokens, always returns success (prevents enumeration)

• Email verification required before full access

• Google OAuth support (optional, requires GOOGLE_CLIENT_ID)

Layer 3: Authorization

RBAC Role-Based Access Control

• Three roles: user, admin, agency

• Admin-only routes gated at both frontend (AdminOnly wrapper) and backend

• Agency dashboard restricted to paid plans (starter, pro, concierge)

• All authenticated endpoints verify JWT before processing

• Feature flags per route module — disabled modules return 503

Layer 4: Input Protection

Validation Input Sanitization

• Request body size limit: 5MB (prevents DoS via large payloads)

• URL validation: blocks private/internal IPs (SSRF protection via is_public_url)

• Email validation: format + domain checks

• Domain validation: rejects invalid/malicious domains

• HTML sanitization: all user-generated content sanitized

• Name sanitization: strips control characters and XSS vectors

• Social handle validation: rejects path traversal and control characters

• Pydantic models enforce type safety on all request bodies

Layer 5: Rate Limiting

SlowAPI Per-Endpoint Rate Limits

• All 83 endpoints have rate limits via SlowAPI

• Auth endpoints: 5-10 requests/minute

• Dashboard/read endpoints: 30 requests/minute

• Audit creation: 10 requests/minute

• Rate limit headers included in responses (X-RateLimit-*)

• Rate limit exceeded → 429 with Retry-After header

Layer 6: Security Headers

Headers Defense in Depth

• Content-Security-Policy: restricts script sources, blocks inline scripts

• X-Frame-Options: DENY (prevents clickjacking)

• X-Content-Type-Options: nosniff (prevents MIME sniffing)

• Referrer-Policy: strict-origin-when-cross-origin

• X-API-Version: 1.9.0 (version tracking)

• CORS: whitelist-only origins (no wildcard in production)

• CSP enforced in: firebase.json, index.html, nginx.conf, csp-config.cjs

Security Checklist

✓ HTTPS enforced (TLS 1.2+)
✓ HSTS header configured
✓ JWT in HttpOnly cookies
✓ No localStorage tokens
✓ Passwords hashed with bcrypt
✓ RBAC (user, admin, agency)
✓ Feature flags per module
✓ Body size limit (5MB)
✓ SSRF protection (URL validation)
✓ Input sanitization (HTML, names, handles)
✓ Rate limiting (all 83 endpoints)
