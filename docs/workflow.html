3 Development Workflow

3.1 Daily Development Cycle

1Start of Day
# Pull latest changes git checkout main git pull origin main # Start backend (auto-reloads on changes) cd backend && source .venv/bin/activate uvicorn server:app --host 0.0.0.0 --port 8001 --reload # Start frontend (HMR on changes) cd frontend && npx vite --port 3000
2Create Feature Branch
# Branch naming convention git checkout -b feature/short-description # New features git checkout -b fix/short-description # Bug fixes git checkout -b refactor/short-description # Refactoring git checkout -b hotfix/short-description # Critical production fixes # Examples git checkout -b feature/add-competitor-export git checkout -b fix/pricing-page-crash git checkout -b hotfix/stripe-webhook-timeout
3Develop & Test Locally
# Write code, then run tests cd backend python -m pytest tests/unit/ -x -q # Backend unit tests (440) python -m pytest tests/integration/ -x -q # Integration tests (188) cd ../frontend npx jest --passWithNoTests # Frontend tests (47) # Run specific test file python -m pytest tests/unit/test_seo_analyzer.py -v npx jest src/__tests__/Landing.test.jsx # Run with coverage python -m pytest tests/unit/ --cov=backend --cov-report=html npx jest --coverage
4Manual Browser Testing

For UI changes, manually test in the browser:

  • Open http://localhost:3000
  • Test the specific page/feature you changed
  • Check browser console for errors (F12 → Console)
  • Test on mobile viewport (F12 → Toggle device toolbar)
  • Test with slow network (F12 → Network → Throttling)
  • Verify all interactive elements work (buttons, forms, links)
5Commit Changes
# Conventional commit format git add . git commit -m "feat: add competitor CSV export button" git commit -m "fix: resolve Pricing.jsx TrendingUp import crash" git commit -m "refactor: extract _store_base_url to dependencies.py" git commit -m "test: add unit tests for social_fetcher retry logic" git commit -m "docs: update README with v1.9.0 changes" git commit -m "chore: bump version to 1.9.0" # Push to remote git push origin feature/add-competitor-export
PrefixUsage
featNew feature or functionality
fixBug fix
refactorCode restructuring without behavior change
testAdding or updating tests
docsDocumentation changes
choreMaintenance, dependencies, config
hotfixCritical production fix

4.2 Running Tests

1Backend Unit Tests
# Run all unit tests cd backend python -m pytest tests/unit/ -x -q # Run with verbose output python -m pytest tests/unit/ -v # Run specific test file python -m pytest tests/unit/test_seo_analyzer.py -v # Run specific test function python -m pytest tests/unit/test_seo_analyzer.py::test_analyze_url -v # Run with coverage report python -m pytest tests/unit/ --cov=. --cov-report=term-missing # Run with parallel execution (faster) python -m pytest tests/unit/ -n auto
2Backend Integration Tests
# Requires MongoDB connection export MONGO_URL="mongodb://localhost:27017/goodly_test" # Run integration tests python -m pytest tests/integration/ -x -q # Run specific integration test python -m pytest tests/integration/test_server_endpoints.py -v
3Frontend Tests
# Run all frontend tests cd frontend npx jest --passWithNoTests # Run with watch mode (auto-rerun on changes) npx jest --watch # Run specific test file npx jest src/__tests__/Landing.test.jsx # Run with coverage npx jest --coverage
4Pre-Commit Test Checklist
○ All backend unit tests pass (440/440)
○ All frontend tests pass (47/47)
○ No new console errors in browser
○ New code has test coverage
○ No linting errors
○ Production build succeeds

5.3 Code Quality Rules

✓ No React hooks after early returns
✓ All lucide-react icons properly imported
✓ dangerouslySetInnerHTML only with HTML escaping
✓ No eval() or new Function()
✓ No innerHTML / outerHTML / insertAdjacentHTML
✓ No bare except: pass without comments
✓ No silent .catch(() => {})
✓ All setTimeout/setInterval properly cleaned up
✓ All addEventListener properly removed
✓ All <img> tags have alt attributes
✓ No hardcoded production URLs in code
✓ No TODOs/FIXMEs/HACKs
✓ Generic error messages (no internal leaks)
✓ All user inputs validated and sanitized

6 Git Workflow & Branching

6.1 Branch Strategy

# Main branches main # Production-ready code staging # Pre-production testing develop # Integration branch (optional) # Feature branches (from main) feature/* # New features fix/* # Bug fixes refactor/* # Code refactoring # Release branches release/* # Release preparation # Hotfix branches (from main) hotfix/* # Critical production fixes

6.2 Pull Request Process

1Create PR
# Push branch and create PR via GitHub CLI gh pr create \ --title "feat: add competitor CSV export" \ --body "## What - Adds CSV export button to competitor comparison page - Exports score, keywords, and gap analysis ## Testing - [x] Unit tests added - [x] Manual browser testing done - [x] All existing tests pass ## Screenshots (attach if UI change)" \ --base main \ --head feature/add-competitor-export
2PR Review Checklist
○ Code follows project conventions
○ All tests pass (CI checks green)
○ New code has test coverage
○ No security vulnerabilities introduced
○ No hardcoded secrets or URLs
○ Error handling is appropriate
○ Documentation updated if needed
○ Version bumped if needed
3Merge Strategy
# Squash merge (recommended for feature branches) # Keeps main branch history clean gh pr merge --squash feature/add-competitor-export # Rebase merge (for clean linear history) gh pr merge --rebase feature/add-competitor-export # After merge, delete the branch git branch -d feature/add-competitor-export git push origin --delete feature/add-competitor-export

18 Troubleshooting Guide

18.1 Common Issues

1Backend won't start
# Check Python version python3 --version # Must be 3.11+ # Check virtual environment which python3 # Should point to .venv/bin/python3 # Check dependencies pip install -r requirements.txt # Check port availability lsof -i :8001 # Should be free # Check .env file exists ls -la backend/.env
2Frontend won't start
# Check Node version node --version # Must be 18+ # Clear node_modules and reinstall rm -rf node_modules package-lock.json npm install # Check port availability lsof -i :3000 # Should be free # Check .env file cat frontend/.env # Should have REACT_APP_BACKEND_URL
3Tests failing
# Check test database curl http://localhost:8001/api/health # Check database status # Run specific failing test with verbose output python -m pytest tests/unit/test_seo_analyzer.py::test_analyze_url -vv # Check for environment issues python -m pytest tests/unit/ -x --tb=long # Clear Python cache find . -type d -name __pycache__ -exec rm -rf {} +
4CORS errors in browser
# Check CORS_ORIGINS includes frontend URL echo $CORS_ORIGINS # For local dev, ensure it includes: CORS_ORIGINS=http://localhost:3000 # Check backend is running curl http://localhost:8001/api/health # Check frontend .env points to correct backend cat frontend/.env | grep REACT_APP_BACKEND_URL
5MongoDB connection issues
# Check connection string format # Must be: mongodb+srv://user:pass@cluster.mongodb.net/dbname # Check network access in Atlas # Atlas → Network Access → Add IP (or allow all: 0.0.0.0/0) # Test connection directly python3 -c " from pymongo import MongoClient client = MongoClient('MONGO_URL', serverSelectionTimeoutMS=5000) client.admin.command('ping') print('Connected!') "
6Stripe webhook not receiving events
# Check webhook endpoint in Stripe Dashboard # Dashboard → Developers → Webhooks → goodly-api # Verify endpoint URL is correct # Must be: https://api.searchgoodly.com/api/billing/webhook # Check webhook secret matches # STRIPE_WEBHOOK_SECRET must match the signing secret in Stripe Dashboard # Test webhook locally stripe listen --forward-to localhost:8001/api/billing/webhook
